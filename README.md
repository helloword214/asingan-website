# Asingan Fire Station Website

## Project Guide

This project aims to create a visually striking and modern fire station website that communicates the station's history, leadership, assets, and public service role while showcasing advanced CSS design, animation, and frontend presentation skills.

Primary project reference:

- [AGENTS](./AGENTS.md)
- [Source of Truth](./docs/SOURCE_OF_TRUTH.md)
- [Prompting Rules](./docs/PROMPTING_RULES.md)

## Core Goal

The website should feel official, informative, trustworthy, and community-centered while also functioning as a strong showcase of frontend design skill.

This project has two main objectives:

- Serve the public with clear and useful information about the fire station.
- Showcase strong CSS, animation, layout, and visual storytelling skills through a premium frontend experience.

## Design Direction

The site should balance two qualities:

- Public-service clarity
- Portfolio-level presentation

The visual direction should be modern, cinematic, and interactive without losing the dignity and professionalism expected from a fire station website.

Recommended design traits:

- Strong visual identity inspired by fire, duty, and service
- High-impact hero section with polished layout and motion
- Purposeful CSS animation such as particles, embers, smoke effects, reveals, and hover interactions
- Smooth scrolling transitions and section-based storytelling
- Clean, readable content areas for important public information
- Mobile-first behavior with good performance and accessibility

## Content Goals

The website should clearly present:

- The history of the fire station
- Fire marshal and key personnel profiles
- Available assets, vehicles, and equipment
- Public services and station responsibilities
- Emergency and contact information
- Real photos of the station, personnel, and assets

## Creative Principle

Animation and visual effects should support the message, not distract from it.

Decorative effects such as particles, layered backgrounds, motion, and transitions should make the website feel alive and memorable, but emergency details and official information must remain easy to read and access.

## Suggested Main Pages

- Home
- History
- Fire Marshal and Personnel
- Assets Available
- Services
- Gallery
- Contact and Emergency Information

## Success Criteria

The project is successful when:

- The website looks credible and professional for a real fire station
- The design clearly reflects strong CSS and frontend execution
- Animations feel intentional and polished
- The content is easy to understand and navigate
- Real station identity is visible through photos and accurate information
- The site works well on mobile and desktop

## Working Standard

Every page should answer both of these questions:

1. Does this help represent the fire station well?
2. Does this show a high level of frontend and CSS skill?

If both answers are yes, the project is on the right path.
